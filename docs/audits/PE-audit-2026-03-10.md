## §17 Walk-Forward Validation — Implementado (2026-03-11)

**Finding F6 cerrado:** Walk-forward validation mínima implementada.

### Archivos creados
- `packages/prediction/src/validation/walk-forward.ts` — `runWalkForward()` pura, sin IO
- `packages/prediction/src/validation/metrics.ts` — Log Loss, Brier, accuracy, draw rate, goals, calibración
- `packages/prediction/src/validation/index.ts` — exports agregados al módulo validation existente
- `scripts/validate-v2-walkforward.ts` — CLI script con V1 comparison cuando disponible

### Comando
```bash
pnpm validate:v2 [COMP] [YEAR]
# Ejemplo:
pnpm validate:v2 PD 2024
```

### Métricas implementadas (§17)
- Log Loss 1X2 (naive baseline ≈ 1.099)
- Multi-class Brier Score (random baseline ≈ 0.667)
- Accuracy (argmax prediction)
- Draw rate: predicha vs real
- Goals: lambda_home+away vs goles reales por partido
- Calibración: 10 buckets, 3 outcomes, mean_predicted_prob vs actual_hit_rate

### Anti-leakage garantías
1. `runWalkForward` usa `sorted.slice(0, i)` — solo partidos estrictamente anteriores
2. El engine aplica segundo filtro interno `utcDate < kickoffUtc`
3. Misma frontera de temporada (1 julio UTC) que `v2-runner.ts`

### V1 comparison
Si existe `cache/predictions/historical-backtest.json` (generado por `run-backtest.ts`), el script carga métricas V1 y compara Log Loss, Brier, Accuracy, draw rate y goles en la misma tabla.
