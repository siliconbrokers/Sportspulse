### Fixture data design

**F1 — Baseline Normal:**
- 4 teams, 20+ finished matches (5 per team), upcoming matches available
- All signals computable, no warnings expected
- Full geometry, deterministic ordering

**F2 — Insufficient History:**
- 4 teams, but one has only 2 finished matches
- INSUFFICIENT_HISTORY warning for that team
- Other teams have full 5-match window

**F3 — No Upcoming Match:**
- 4 teams, one has no scheduled match
- NO_UPCOMING_MATCH + MISSING_SIGNAL warnings
- nextMatch: undefined for that team

**F4 — Partial Snapshot:**
- Teams with some matches but one team has 0 matches
- MISSING_SIGNAL for both signals on that team
- Snapshot still builds

**F5 — Stale Fallback:**
- Not a fixture with static data per se
- context.json indicates this is a stale test scenario
- Expected: STALE_DATA + PROVIDER_ERROR warnings
- Tested via SP-0902 runner with special setup

**F6 — Layout Degenerate:**
- All teams have 0 finished matches, no upcoming matches
- All layoutWeight = 0
- LAYOUT_DEGRADED warning
- Equal-area geometry fallback

### Generating expected outputs
Run buildSnapshot with fixture inputs and freeze the output as expected.snapshot.json.
For signals: extract signal DTOs from the built snapshot.
