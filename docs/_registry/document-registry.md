## Governance Rules for This Registry

1. **New artifact created** → add row to appropriate section
2. **Artifact renamed** → update canonical_path, keep artifact_id stable
3. **Artifact superseded** → change status to `superseded`, add `superseded_by`, move to archive section
4. **Artifact deleted** → only if non-canonical junk — add row to archive with reason
5. **Never** delete a row without replacement
